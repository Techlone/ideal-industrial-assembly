1. Основные положения
- изменение рецептов в сборке ИИС осуществляется с помощью мода MineTweaker 3
- версия MineTweaker 3 является доработанной под нужды ИИС, все изменения приводятся по адресу https://github.com/Techlone/MineTweaker3
- рабочей папкой принято считать каталог "./minecraft/scripts"
- скриптом является файл с расширением ".zs" и содержащим код на языке ZenScript
- новые рецепты предметов располагаются в отдельном скрипт-файле мода

2. Наименование и расположение файлов внутри рабочей папки
- все скрипт-файлы модов находятся непосредственно в рабочей папке
- скрипт-файл мода именуется аналогично строковому идентификатору мода без пробелов, спецсимволов и дополнительных сокращений, каждое слово начинается с заглавной буквы (например "AppliedEnergistics2.zs")
2.1 Подпапки
- скрипт-файлы в подпапках именуются по усмотрению
- папка ./fixes используется для скриптов, которые одинаково изменяют большие списки предметов (примеры в этой же папке)
- папка ./disabled является хранилищем неиспользуемых скрипт-файлов

3. Стиль написания кода на ZenScript
3.1 Переменные
- объявление новой переменной начинается с ключевого слова "var"
- при наименовании переменных следует использовать lowerCamelCase стиль
- имя переменной должно однозначно определять хранящиеся в этой переменной данные (см. п.3.3)
3.2 Предметы  жидкости и словари
- идентификатор предмета/жидкости/словаря соответствует принятому в игре Minecraft (например "<minecraft:stone>", "<liquid:water>", "<ore:ingotIron>")
- все без исключения идентификаторы перед использованием необходимо определить в переменные
3.3 Рекомендации по наименованию переменных
- префиксом мода является принятая аббревиатура данного мода (см. п.4)
- имена переменных, определеяющих предметы, должны содержать префикс мода, коротый добавляет данный предмет
- имена переменных, определеяющих жидкости, должны содержать префикс "molten" для расплавленных твердых веществ (в основном это металлы) или "liquid" в остальных случаях
- имена переменных, определеяющих словари, должны содержать префикс "ore"
3.3.1 Строковые идентификаторы
- имя переменной должно максимально совпадать с идентификатором предмета/жидкости/словаря
3.3.2 Числовые идентификаторы и идентификаторы с метадатой
- имя переменной должно максимально совпадать с именем предмета/жидкости
3.3.3 Списки и массивы
- одномерные массивы или списки именуются по общим правилам переменных с добавлением постфикса "List"
- допускается добавление окончания -s (множественое число в английском языке (с) Капитан О) вместо постфикса "List"
- многомерные массивы именуются по усмотрению с учетом всех рекомендаций
